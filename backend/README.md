-- Vi har valt FastAPI som backend eftersom det gör det lätt att bygga tydliga API-anrop mellan frontend och databasen.
-- Vi använder backend som ett mellanlager så att frontend inte pratar direkt med databasen.
-- Det gör systemet säkrare eftersom databaslösenord och hemliga nycklar stannar i backend.
-- Backend tar emot anrop från frontend, kontrollerar dem och skickar sedan rätt data till eller från databasen.
-- Vi använder JWT för att hålla reda på vem som är inloggad.
-- Det gör att skyddade delar av systemet bara kan användas av rätt användare.
-- Vi har också lagt viss logik i backend, till exempel kontroll av data och koppling mellan poster och begrepp.
-- AI-funktionen ligger också i backend så att API-nyckeln inte syns i webbläsaren.
-- Vi tycker därför att backend är ett bra mellanlager mellan användaren, systemet och databasen.
-- Databasen är fortfarande kärnan, men backend gör den lättare att använda på ett säkert och tydligt sätt.
